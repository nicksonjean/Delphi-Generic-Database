Frequently Asked Questions  

This page provides answers to frequently asked questions about Devart dbExpress drivers. If you haven not found an answer to your question, please see the Support file.

 

Licensing and Subscriptions
  1. Am I entitled to distribute applications written with Devart's dbExpress drivers?

    If you have purchased a full version of the driver, you are entitled to distribute the driver with pre-compiled programs that use the driver. You are not entitled to sell any components or libraries inherited from the driver or using its source code. For more information, see the License.rtf file in the installation directory of the driver.
  2. I have a registered version of a driver. Will I need to pay to upgrade to future versions?

    Starting with dbExpress Driver for Oracle 4.00, dbExpress Driver for SQL Server 4.00, dbExpress Driver for MySQL 4.00, dbExpress Driver for InterBase and Firebird, and dbExpress Driver for PostgreSQL 1.00, users can upgrade to newer versions for free provided they have an active subscription.

    Users who have a license for an earlier version of the driver will have to first upgrade their version to enjoy the benefits of the Subscription Program.

  3. What are the benefits of the Subscription Program?

    The Subscription Program is an annual maintenance and support service for users of Devart dbExpress drivers.

    Users with a valid subscription get the following benefits:

    • Technical support through the Priority Support Program
    • Access to new versions of dbExpress drivers when they are released
    • Access to all updates and hotfixes for dbExpress drivers
    • Notifications of new product versions

    Priority Support is an advanced product support service that provides individual assistance with dbExpress driver-related questions from the developers. Priority support is provided over email and has a two business day response policy.

    The Subscription Program is available for users with a valid subscription, starting with dbExpress Driver for Oracle 4.00, dbExpress Driver for SQL Server 4.00, dbExpress Driver for MySQL 4.00, dbExpress Driver for InterBase and Firebird 2.00, and dbExpress Driver for PostgreSQL 1.00.
  4. Can I use my version of the driver after my subscription expires?

    Yes, you can. dbExpress Driver license is perpetual.
  5. I want a driver subscription! How can I get one?

    An annual driver subscription is included when ordering or upgrading to any non-trial version of dbExpress drivers, starting with dbExpress Driver for Oracle 4.00, dbExpress Driver for SQL Server 4.00, dbExpress Driver for MySQL 4.00, dbExpress Driver for InterBase and Firebird 2.00, and dbExpress Driver for PostgreSQL 1.00.

    You can renew your driver subscription on the Ordering page. For more information, please contact us at sales@devart.com.
  6. Does this mean that if I upgrade to dbExpress Driver for Oracle 4 from version 3, I'll get an annual dbExpress Driver subscription for free?
    Yes. You will also get an annual subscription for free if you upgrade to dbExpress Driver for SQL Server 4, dbExpress Driver for MySQL 4, or dbExpress Driver for InterBase and Firebird 2.
  7. How do I upgrade to a newer version of the driver?

    To upgrade dbExpress drivers, starting with dbExpress Driver for Oracle 4.00, dbExpress Driver for SQL Server 4.00, dbExpress Driver for MySQL 4.00, and dbExpress Driver for InterBase and Firebird 2.00, visit the Customer Portal. For more information, please contact us at sales@devart.com.
General Questions
  1. What software should be installed on the client computer so that my applications that use Devart's dbExpress drivers can run?

    First of all, you should have the driver itself installed on the client computer. Other required sofware depends on the driver:

    dbExpress Driver for Oracle: No additional files are required in the Direct mode. Otherwise, you should install the Oracle client.

    dbExpress Driver for SQL Server: You should have the OLE DB provider installed on the client computer. If you are using Windows 2000 or later, OLE DB is already included. For Windows versions prior to Windows XP, it is highly recommended to download the latest version of Microsoft Data Access Components (MDAC).
    If you work with SQL Server Compact Edition, the server itself should be installed on the client computer.
    Support for some features of SQL Server 2005 like MARS requires Microsoft SQL Server Native Client. If your application has to work with SQL Server 2005, we strongly recommend that you have SQL Native Client installed.

    dbExpress Driver for MySQL: In most cases, no additional files required on the client computer. You need additional files in the following cases:

    • If you are using MySQL Embedded, you need the server itself (libmysqld.dll) and related files, such as errmsg.sys.
    • If you are using the driver in the Client mode, you need the client library (libmysql.dll).

    dbExpress Driver for InterBase and Firebird: You should install the InterBase or Firebird client.

    dbExpress Driver for PostgreSQL: No additional files are required.
  2. I would like to develop an application that works with a database. Should I use data access components or dbExpress drivers?

    The dbExpress technology provides a rather uniform way to access different servers (Oracle, SQL Server, MySQL, etc). It is based on drivers that include server-specific features. Like any universal tool, in many specialized cases dbExpress providers lose some functionality. For example, the dbExpress design-time is quite limited and cannot be enhanced with additional features.

    Our Data Access Components (DACs) are specialized sets of components for specific servers, which have advanced server-specific design-time editors and a component interface similar to that of BDE.

    We aim to support all server-specific features both in dbExpress drivers and DACs. However, the dbExpress technology has some limitations that cannot be overcome.

    Both DACs and dbExpress drivers are based on the same kernel code, and thus have similar performance. In some cases dbExpress is slower because the transmitted data is converted to correspond to the dbExpress standard.
  3. There are several dll files in the installation directory of the driver. Which one should I use in my application?

    It depends on the version of your IDE:

    • For RAD Studio 10.3 and higher, you should use the dbexpXda41.dll file.
    • For RAD Studio 2007 up to RAD Studio 10.2, you should use the dbexpXda40.dll file.
    • For Delphi 2006 and C++ Builder 2006, you should use the dbexpXda30.dll file.
    • For Delphi 6, C++ Builder 6, and Delphi 7, you should use dbexpXda.dll.
    The X letter in the file name depends on your driver.
  4. How can I set the port number for the database connection?

    dbExpress Driver for MySQL, dbExpress Driver for PostgreSQL: You can set the port number in two ways:

    • after the server name, delimited by a colon: SQLConnection.Params.Add('HostName=MyServer:3306');
    • using the 'Server Port' parameter: SQLConnection.Params.Add('Server Port=3306');
      This way works for Delphi 7 and later versions.

    dbExpress Driver for InterBase and Firebird: You can set the port number in the Database parameter in the following format: HostName/Port:DBPath
  5. When I try to run my application that uses Devart dbExpress driver, I get the "Unable to load dbexp*.dll" message.
    The dbexp*.dll file is the driver itself. This file should be located in a folder accessible by your application. When you deploy the application, copy dexp*.dll to the directory specified in the PATH system variable, or to the directory that contains your application.
    The full name of the dexp*.dll file depends on your driver, and your IDE.
